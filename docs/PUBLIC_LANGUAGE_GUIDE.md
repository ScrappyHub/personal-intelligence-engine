# PIE Public Language Guide v1

PIE should be described publicly as:

- local-first AI runtime
- offline-capable personal AI engine
- private project-aware AI workbench
- local model orchestration layer
- reproducible AI execution system
- preview / alpha / core runtime

Avoid public-facing internal build language:

- Tier-0
- FULL_GREEN
- internal witness terms unless in developer docs
- ecosystem-only dependency language

Public promise:

PIE helps users run local AI, manage memory by project, choose models by intent, and preserve private work locally.
